# 企业微信批量转发 RPA 用户说明

## 使用前准备

1. 打开并登录企业微信。
2. 进入包含待转发消息的会话。
3. 手动进入消息多选并勾选本次需要转发的全部消息。
4. 双击 `run-gui.bat`。

## GUI 参数

- **发送数量**：本次最多处理的会话总数。
- **batch_size**：每批会话数，最大为 9。
- **间隔秒**：相邻批次之间的等待时间。
- **哨兵边界**：启用后填写哨兵会话名称；多个名称用逗号分隔。
- **Dry-run 自检**：不点击最终发送按钮。
- **真实发送**：会实际发送消息，必须完成两个确认项。

## 运行步骤

1. 填写参数并选择运行模式。
2. 点击“检查环境”。
3. 确认弹窗提示已识别到正确数量的源消息和“逐条转发”按钮。
4. 点击“启动运行”。
5. 真实发送模式输入 `SEND`。
6. 等待中文完成弹窗，查看计划数量、实际发送数量和总耗时。

修改参数后需要重新点击“检查环境”。

## 程序行为

- 企业微信窗口会自动激活并最大化。
- 每批打开收件人弹窗后，拖动会话列表滚动条到底部。
- 程序会截图确认拖动生效，并确认列表稳定在底部。
- 从底部向上选择会话，每批最多 9 个。
- 遇到哨兵时，只发送哨兵下方的会话。
- “立即停止”只结束当前任务，GUI 保持打开。
- 重新运行会再次从列表底部开始。

## 运行文件

- `logs/wecom_rpa.log`：详细日志和步骤耗时。
- `logs/run_snapshots/*.json`：本次参数快照。
- `screenshots/checkpoints/`：过程截图。
- `screenshots/errors/`：错误截图。

程序不使用 YAML、CSV 或 SQLite。
