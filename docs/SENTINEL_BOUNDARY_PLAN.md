# 哨兵边界实现说明

## 用途

程序从列表底部向上选择会话，遇到用户指定的哨兵后，只发送哨兵下方的会话。

## 输入

- GUI：勾选“启用哨兵”，多个名称用逗号分隔。
- CLI：重复传入 `--sentinel-name`。

启用哨兵但未填写名称时，参数校验失败。

## 批次处理

每批先勾选计划数量的会话，再通过 PaddleOCR 读取左侧已勾选会话名称。内部顺序统一为从底部到顶部。

出现首个哨兵时：

1. 保留哨兵下方的会话。
2. 取消哨兵本身。
3. 取消哨兵上方所有已选会话。
4. 重新扫描蓝色复选框并 OCR 复核剩余会话。
5. 剩余数量为 0 时关闭收件人弹窗并结束。
6. 剩余数量大于 0 时发送尾批，然后结束任务。

示例：

```text
从底部到顶部：群A、群B、群C、哨兵、群D、群E
最终发送：群A、群B、群C
取消：哨兵、群D、群E
```

## 安全条件

以下任一情况都会停止，不点击最终发送：

- OCR 没有读取到已选会话。
- 已识别会话数量与蓝勾数量不一致。
- 存在无法识别名称的会话行。
- 无法点击需要取消的复选框。
- 截断后剩余数量不符合预期。
- 截断后仍检测到哨兵。

## 重启行为

程序不保存发送状态，也不做断点续跑。中断后重新运行会再次从会话列表底部开始，用户应先根据企业微信聊天记录确认是否需要重新运行。
