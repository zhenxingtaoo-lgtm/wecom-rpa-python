# 企业微信桌面端批量转发 RPA 设计文档

## 1. 目标

在 Windows 企业微信桌面端执行数量驱动的批量逐条转发。用户负责在运行前勾选源消息，程序负责：

- 检查并记录源消息与“逐条转发”按钮位置；
- 分批进入收件人选择弹窗；
- 把会话列表滚动到底部；
- 从底部向上勾选最多 9 个会话；
- 根据哨兵边界截断尾批；
- 发送并保存日志、参数快照和截图证据。

## 2. 当前边界

- 不读取 YAML、CSV 或 SQLite。
- 不按群名搜索目标会话。
- 不保存发送状态，也不进行断点续跑。
- 不过滤员工、机器人或群聊类型。
- 源消息数量由检查阶段实际识别结果决定，不固定为 2 条或 2–5 条。
- 中断后再次运行会重新从会话列表底部开始。

## 3. 主要模块

- `config.py`：运行时参数对象、默认值和安全校验。
- `gui.py`：参数输入、环境检查、运行快照、工作线程和中文结果弹窗。
- `forward_flow.py`：批次拆分、源消息重选、收件人选择、哨兵截断和发送复查。
- `screen.py`：截图、PaddleOCR、OpenCV 复选框识别。
- `wecom_window.py`：窗口定位、最大化、激活、点击、拖动和按键。
- `safety.py`：急停控制和每批最多 9 个会话限制。

## 4. 运行参数

用户每次运行前提供：

- `send_count`：计划处理的会话总数，必须大于 0。
- `batch_size`：每批数量，范围为 1–9。
- `batch_interval_sec`：批次间隔，不能为负数。
- `dry_run`：Dry-run 或真实发送。
- `sentinel_enabled`：是否启用哨兵边界。
- `sentinel_names`：一个或多个哨兵名称。

内部固定策略：

- 企业微信窗口标题为“企业微信”。
- 收件人模式为 `bottom_of_picker`。
- 选择方向为 `bottom_to_top`。
- OCR 使用 PaddleOCR 中文模型，正常空结果不回退 Windows OCR。
- 急停快捷键默认为 `Ctrl+Alt+Q`。

## 5. GUI 检查阶段

参数变化会使之前的检查结果失效。点击“检查环境”后：

1. 校验发送数量、批次大小、间隔、哨兵和真实发送确认。
2. 定位、激活并最大化企业微信窗口。
3. 保存全屏检查截图。
4. 扫描源消息区域中的蓝色复选框。
5. 按同一 X 列聚类，排除标题图标和底部工具栏蓝色图标。
6. 使用 PaddleOCR 查找“逐条转发”文字并记录点击坐标。
7. 将源消息 X/Y 比例和按钮比例写入本次内存配置。
8. 检查失败时弹窗并返回 GUI；成功时恢复 GUI 前台并显示识别数量。

检查阶段未识别到任一源消息或逐条转发按钮时，不允许启动运行。

## 6. 批次流程

真实发送每批执行：

1. 定位并激活企业微信窗口。
2. 第一批复核当前源消息；后续批次通过右键菜单重新进入多选并勾选记录的源消息行。
3. 复核源消息数量与检查阶段一致。
4. 点击检查阶段记录的“逐条转发”坐标。
5. OCR 确认“发送给”弹窗已出现。
6. 拖动左侧会话列表滚动条到底部并验证。
7. OpenCV 识别当前可见灰色会话复选框。
8. 从底部向上点击本批需要的会话。
9. 扫描蓝色复选框，复核实际勾选数量。
10. 启用哨兵时 OCR 读取已选会话名称并执行截断。
11. 点击最终发送按钮。
12. 保存发送后截图，并确认收件人弹窗已关闭。

## 7. 滚动到底部

滚动策略不随会话总数线性增长：

1. 截取左侧会话列表区域作为滚动前基准。
2. 鼠标移动到滚动条顶部滑块附近并短暂停留，让细滚动条显示。
3. 按住左键，将滑块拖到轨道底部。
4. 截图比较拖动前后列表画面，差异不足说明拖动无效。
5. 提供三个相邻起点尝试；全部无效时立即停止。
6. 拖动成功后点击底部轨道并截图复核。
7. 连续两轮画面差异为 0 时，确认列表已稳定在底部。

底部轨道点击只负责验证，不承担把列表逐页滚到底的职责。

## 8. 源消息重选

- 检查阶段记录所有识别到的源消息行。
- 后续批次优先复用已记录的右键菜单位置。
- 进入多选后先识别当前已选源消息，再点击尚未选中的记录行。
- 最终复核数量和位置必须与检查阶段一致。
- 数量发生变化或任一点击无法确认时停止。

## 9. 哨兵边界

已选会话按从底部到顶部排序。出现首个哨兵时：

- 保留哨兵下方会话；
- 取消哨兵本身及其上方会话；
- 再次 OCR 和蓝勾复核剩余会话；
- 剩余数量为 0 时关闭弹窗并结束；
- 剩余数量大于 0 时发送尾批，然后结束任务。

名称识别失败、数量不一致、存在未知行或截断复核失败时停止。

## 10. Dry-run 与真实发送

Dry-run：

- 执行参数校验、窗口定位、截图和批次模拟；
- 不点击最终发送按钮；
- 返回实际发送数量 0。

真实发送：

- CLI 必须同时提供 `--no-dry-run --real-send --i-understand-this-will-send-messages`。
- GUI 必须选择真实发送、勾选两个确认项并输入 `SEND`。

## 11. 停止与结果

- GUI“立即停止”只结束当前任务，不关闭窗口。
- 当前安全步骤结束后流程返回 GUI。
- 成功或停止时，中文弹窗显示计划处理数、实际发送数和总耗时。
- 失败时，中文弹窗显示总耗时和错误原因。

## 12. 运行留痕

- 主日志记录每一步开始、完成、坐标、识别结果和耗时。
- `logs/run_snapshots/*.json` 保存最终运行参数和检查阶段坐标。
- `screenshots/checkpoints/` 保存关键步骤截图。
- `screenshots/errors/` 保存失败截图和原因。

## 13. 安全停止条件

- 找不到企业微信窗口。
- 源消息或逐条转发按钮未识别。
- 源消息数量与检查记录不一致。
- 收件人弹窗未出现或意外关闭。
- 滚动条拖动无效或无法证明已到底。
- 灰色复选框数量不足。
- 勾选后蓝色复选框数量不一致。
- 哨兵识别或截断复核失败。
- 发送后截图不可用或弹窗仍存在。
- 急停触发。
